# memory

一个面向个人知识整理与复用的 **知识库 MVP**。  
项目的目标是把分散的笔记、访谈记录、网页内容和参考资料整理成可复用的知识卡片，并进一步围绕这些资料进行提问、总结和 PRD 提纲生成。

## 项目简介

`memory` 是一个以 **“知识沉淀 + 工作台生成”** 为核心流程的轻量原型项目。

用户可以先在知识库中沉淀内容，再把选中的知识、网页资料或本地文件加入工作台，基于这些资料进行：

- 问答
- 总结生成
- PRD 提纲生成

整体更偏向 **产品经理 / 用户研究 / 内容整理** 场景下的知识加工与复用。

---

## 主要功能

### 1. 知识库管理
- 新增知识卡片
- 按主题浏览知识
- 按标题、摘要、内容、标签进行搜索
- 查看知识详情
- 删除知识

### 2. AI 整理知识
新增知识时，内容会先经过 AI 整理，再生成更适合沉淀的：
- 标题
- 摘要
- 主题
- 标签

这样可以把原始输入整理成更适合后续复用的知识卡片。

### 3. 工作台资料组装
工作台支持把不同来源的资料组合起来作为上下文：
- 知识库中的已有知识
- 本地资料（当前版本只记录文件基础信息，不上传文件内容）
- 网页链接（服务端抓取网页正文，并自动生成摘要与标签）

### 4. 基于资料进行问答
在工作台中，可以围绕当前已选资料直接提问。  
回答默认优先基于当前资料生成，在开启“允许联网补充”后，也可以在资料不足时结合外部信息补充回答。

### 5. 生成总结
基于当前选中的资料，生成一版轻量总结，适合快速回顾和整理要点。

### 6. 生成 PRD 提纲
基于当前选中的资料，生成 PRD 提纲，帮助把访谈、知识卡片和参考资料进一步转成结构化输出。

### 7. 本地持久化
当前版本的数据主要面向本地演示与原型验证。  
知识列表和工作台资料会保存在浏览器本地存储中，在同一浏览器环境下刷新页面后仍可保留。

## 适用场景

这个项目更适合以下场景：

- 用户访谈记录整理
- 产品需求背景沉淀
- PRD 前期资料汇总
- 个人知识卡片管理
- 网页资料快速归纳
- 基于已有资料做问答和生成
